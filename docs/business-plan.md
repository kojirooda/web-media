# 事業計画: AI駆動ビジネスで月100万円達成
> 策定日: 2026-03-16 | 目標達成期限: 2026-09-16

---

## エグゼクティブサマリー

**目標**: 6ヶ月で月次売上100万円以上
**投入リソース**: Claude Code（主役）+ 人間（日次30分チェックのみ）
**初期投資**: 最大100万円（段階的執行）
**月額コスト**: 売上連動型（収益の15-20%以内に収める）

---

## 現状分析

### 強み（使えるもの）
- Claude Code による高速開発・コンテンツ生成能力
- 40代経営者・3児の父という高信頼性ブランドボイス
- web-mediaリポジトリ（SEOコンテンツ基盤）が稼働中
- 「40代 副業」というニーズの高いジャンルで既にコンテンツあり

### 課題認識
- SEO単体では6ヶ月で月100万円は非現実的（通常12-18ヶ月）
- コンテンツ事業のみでは収益化が遅い
- **解決策**: SEOを集客装置として使い、高単価B2Bと定額SaaSで速攻収益化

---

## 3層ビジネスモデル（並行展開）

```
Layer 1: B2B AIコンサル受託（即収益）        → Month 1-2 から黒字
Layer 2: AI SaaS プロダクト（安定収益）      → Month 3-4 から本格始動
Layer 3: SEOメディア × アフィリエイト（複利）→ Month 6 以降に本格化
```

### Layer 1: AI業務自動化コンサル（B2B受託）

**ターゲット**: 中小企業経営者（従業員5-50名）
**提供価値**: Claude Codeで業務自動化を構築・納品する
**価格体系**:
- 初期構築費: ¥300,000〜500,000（1件）
- 月次保守: ¥50,000〜100,000/月

**Claude Codeの役割**:
- 提案書・見積書の自動生成
- 実際の自動化システム構築（80-90%）
- 納品レポート・マニュアル生成
- 月次レポート自動送信

**月次収益試算**:

|---|---|---|---|
| M1 | 1件×¥400K | - | ¥400K |

### Layer 2: AI SaaS「BizAI」（中小企業向け）

**プロダクト名（仮）**: BizAI - 中小企業のAI業務アシスタント
**コアコンセプト**: 経営者が毎日使う業務文書をAIで自動化

**機能セット（Claude API powered）**:
1. SNS投稿文生成（Instagram/X/LINE）
2. メルマガ・ニュースレター生成
3. 求人票・採用文書生成
4. 議事録・報告書自動作成
5. 顧客対応メールテンプレート生成

**価格**:
- スタンダード: ¥9,800/月
- ビジネス: ¥19,800/月（推奨）
- エンタープライズ: ¥49,800/月

**収益達成シナリオ**（ビジネスプラン中心）:

### Layer 3: SEOメディア（既存web-media強化）

**戦略転換**: 純アフィリエイトから「SaaSへの集客ファネル」へ

**コンテンツ戦略**:
- メインKW: 40代 副業、40代 AI活用、中小企業 AI導入
- 記事生成: Claude Code が週5本自動生成・スケジュール投稿
- アフィリエイト: AIツール系（Claude Pro, Notion, etc.）

**Claude Codeの役割**:
- 毎日キーワード調査・記事プランニング
- 記事執筆（アウトライン→本文→SEOチェック）
- 内部リンク最適化
- GitHub Actionsで自動デプロイ

**収益試算**（M6時点）:
- アフィリエイト収益: ¥50,000〜200,000/月
- SaaSへの送客: 月10-20人（Layer 2 に貢献）

## 人間の役割（日次30分以内）

### 毎日やること（15-20分）
1. 自動生成された日次レポートを確認（Claude Codeが送付）
2. 当日公開予定コンテンツの最終確認・承認（または完全自動化）
3. 新規顧客・問い合わせのエスカレーション対応

### 週次やること（1-2時間）
1. 週次KPIダッシュボード確認
2. 新規コンサル案件の対面打ち合わせ（Claude Codeが資料準備）
3. 方向感の微調整・優先度変更の指示

### Claude Codeがやること（残り99%）
- 記事生成・公開・SEO最適化
- SaaS機能開発・バグ修正
- 顧客サポートの一次対応（FAQ自動返答）
- 月次・週次・日次レポート生成
- 請求処理・Stripe Webhook処理
- 新規顧客オンボーディングメール送信
- SNS投稿スケジューリング
- コンサル提案書・報告書作成

## リスクと対策

| リスク | 確率 | 対策 |
|---|---|---|
| SaaS顧客獲得が遅い | 中 | Layer 1 受託でキャッシュ確保しながら並行 |
| Claude API コスト超過 | 低 | 利用量モニタリング自動化・上限設定 |
| SEOが想定より遅い | 中 | メディアへの依存度を下げ、SNS・広告を補完 |
| コンサル案件が取れない | 低 | 既存人脈（経営者ネットワーク）から初期受注 |
| 競合SaaSの台頭 | 中 | ニッチ特化・ブランドボイスによる差別化 |
