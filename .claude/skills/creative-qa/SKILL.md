# Creative QA スキル

記事の品質チェックを実行します。以下の観点で評価し、**Critical / Important / Nice to have** の3段階で報告してください。

## 評価基準

### 🔴 Critical（公開ブロック）
これが1つでもあれば公開不可。即時修正が必要。

- [ ] 事実誤認・誤情報が含まれている
- [ ] 引用・数値の出典が不明または不正確
- [ ] SEOルール違反: H1 が複数ある、または存在しない
- [ ] meta description が未設定または120文字超
- [ ] 構造化データ（Article schema）が未実装
- [ ] ブランドボイスと著しく乖離している（docs/brand-voice.md 参照）

### 🟡 Important（公開前に修正推奨）
品質に直接影響するが、緊急ではない。

- [ ] 内部リンクが3本未満
- [ ] H2/H3 にターゲットキーワードが含まれていない
- [ ] 導入部で読者の悩みが言語化されていない
- [ ] 具体的な数値・体験談が少ない（抽象的な記述が多い）
- [ ] CTAまたは次のアクション提示がない
- [ ] 画像の alt テキストが未設定

### 🟢 Nice to have（改善提案）
あると良いが必須ではない。

- [ ] 関連キーワードの自然な組み込み
- [ ] 読みやすさ改善（1文が長すぎる箇所）
- [ ] FAQ セクションの追加検討
- [ ] SNS シェアを促す要素

## 出力フォーマット

チェック完了後、以下の形式で報告してください：

```
## QA レポート: [記事タイトル]
実行日時: [日時]

### 🔴 Critical (X件)
- [問題の説明と修正箇所]

### 🟡 Important (X件)
- [問題の説明と改善案]

### 🟢 Nice to have (X件)
- [改善提案]

### 総合評価
[公開可 / 修正後公開可 / 公開不可] + コメント
```

## チェック実行手順

1. `CLAUDE.md` の SEO ルールを参照
2. `docs/brand-voice.md` のブランドボイスと照合
3. 対象記事ファイルを全文読み込み
4. 上記チェック項目を順番に評価
5. レポートを出力
