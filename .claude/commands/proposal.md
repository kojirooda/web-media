# /proposal コマンド — AI業務自動化 コンサル提案書の自動生成

## 使い方
```
/proposal [顧客名] [業種] [主な課題をカンマ区切りで]
```

**例**:
```
/proposal "田中商店" "小売・雑貨" "SNS更新が週1回,在庫管理が手動,スタッフ採用文書に時間がかかる"
/proposal "鈴木法律事務所" "士業・法律" "顧客対応メールに時間がかかる,請求書作成が月末に集中,ブログ更新ができていない"
/proposal "ABC不動産" "不動産" "物件紹介文作成に時間がかかる,問い合わせ返信が遅い,SNS更新が止まっている"
```

引数なしで実行した場合は、以下の情報を対話形式で質問してから生成すること。

---

## 実行手順

### STEP 1: 情報収集

以下を確認・整理する:

```
1. 顧客基本情報
   - 会社名 / 屋号
   - 業種（飲食・小売・士業・不動産・製造・サービス等）
   - 従業員規模（1人・2-5人・6-20人・21-50人）
   - 代表者名（わかれば）

2. 課題（ヒアリング内容または引数から）
   - 最重要課題（1〜2つ）
   - 副次的な課題（3〜5つ）
   - 現状の工数・時間（概算でよい）

3. 提案方向性
   - 業種に合った最適な自動化領域を判断
   - フェーズ構成を決定（2〜3フェーズ）
   - 費用感の方向性（小規模: 15-30万 / 標準: 30-50万 / 大型: 50-100万）
```

### STEP 2: 業種別テンプレート選択

業種に応じて以下の自動化領域を組み合わせる:

| 業種 | 優先自動化領域 |
|---|---|
| 飲食・カフェ | SNS投稿、予約DM対応、メニュー告知文、スタッフマニュアル |
| 小売・EC | 商品説明文、SNS投稿、在庫アラート、顧客メール |
| 士業（士業・法律・税理士） | 顧客対応メール、請求書・見積書、業務報告書、ブログ |
| 不動産 | 物件紹介文、問い合わせ対応、SNS投稿、月次報告書 |
| 製造・卸 | 見積書・提案書、社内報告書、採用文書、展示会資料 |
| サービス業全般 | SNS投稿、メルマガ、顧客フォローメール、業務マニュアル |

### STEP 3: 提案書生成

`docs/consulting/proposal-template.md` を読み込み、以下のルールで生成する:

**金額設定基準**:
```
従業員1-5人   → 初期: 15-30万円、月次: 2-3万円
従業員6-20人  → 初期: 30-50万円、月次: 3-5万円
従業員21-50人 → 初期: 50-100万円、月次: 5-10万円
```

**フェーズ構成**:
```
フェーズ数: 2〜3
期間: 全体で6〜8週間
優先度の高い課題から着手する順番で設計
```

**ROI試算の基準**:
```
代表者の時給換算:
  小規模事業者 → ¥3,000/時間
  中小企業経営者 → ¥5,000/時間
  専門職（士業等）→ ¥8,000/時間
```

**文体・トーン**（docs/brand-voice.md 参照）:
- 「です・ます」調
- 押しつけがましくなく、先輩経営者として寄り添う
- 数字・具体例を必ず含める
- 「絶対」「必ず」等の断言は避ける

### STEP 4: 出力・保存

生成した提案書を以下のパスに保存する:
```
docs/consulting/proposals/proposal-{顧客名}-{日付}.md
```

保存後、以下を出力する:
```
✅ 提案書を生成しました
📄 保存先: docs/consulting/proposals/proposal-{顧客名}-{日付}.md

---
【提案サマリー】
顧客: {顧客名}（{業種}）
提案フェーズ: {フェーズ数}フェーズ / {週数}週間
初期費用: {金額}
月次保守: {金額}/月
想定削減時間: {時間}時間/月
ROI回収期間: {月数}ヶ月

---
次のステップ:
- 提案書を送付する前に内容を確認してください
- 修正が必要な箇所があれば「この部分を〜に変えて」と指示してください
- PDFに変換したい場合は /publish proposal コマンドを使用してください
```
